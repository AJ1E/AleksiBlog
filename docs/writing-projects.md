# 项目展示写作与封面规范

本手册用于以后新增或修改 `src/content/projects/` 下的项目。项目页既是作品记录，也是网站长期维护的一部分；优先保持内容清楚、封面有辨识度、页面可读。

## 内容与命名

- 一个项目对应一个 `src/content/projects/<stable-slug>.mdx` 文件；不要为了改标题而改 slug。
- 项目名称、标题、摘要、分类、标签、正文和站内链接文案默认使用中文；技术名词可保留必要英文。
- 填写准确的 `year`、`status`、`category`、`tags`、`accent`、`pattern` 和 `order`，再运行 `pnpm build` 验证 schema。
- 修改既有项目正文时，如内容实质更新，应同步更新 `updatedAt`。

## 封面与动效

- 每个项目必须有与主题相符、可区分的封面 SVG 场景，由 `src/components/ProjectVisual.astro` 维护。
- 封面左上角使用简短英文缩写或单词，例如 `DASH`、`CMS`、`OPS`、`BLOG`；封面内部不得出现中文。
- 封面左下角使用简短英文说明，描述该项目的技术或主题，不写长句和私人信息。
- 项目卡片在鼠标悬停和键盘聚焦时应有轻量、与主题匹配的动效。优先使用现有 CSS/SVG，不引入 GIF、JavaScript 动画库或额外依赖。
- 每种动效都必须保留 `prefers-reduced-motion` 降级，且不可造成文字遮挡、布局跳动或移动端溢出。

## 详情页缩写说明

- 每个项目详情页标题区的第一行必须解释封面左上角缩写。
- 说明顺序固定为：`缩写 · 英文全称 · 中文含义`。
- 示例：`OPS · Operations · 本地服务与监控`。
- 新项目加入新缩写时，同时更新 `src/pages/projects/[slug].astro` 内的缩写映射；不要让封面缩写与详情页说明脱节。

## 发布前检查

1. 本地打开项目列表和项目详情页，检查封面、缩写说明、标题、状态、标签和链接。
2. 检查桌面、移动端和深色模式；卡片与详情页文字不得重叠或溢出。
3. 运行 `pnpm build`，再检查 `git diff --check` 与完整 diff。
4. 任何封面图片或截图都不得包含凭据、真实服务器地址、私有面板、私人路径或个人敏感信息。
