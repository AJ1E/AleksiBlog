# 安全维护与依赖升级

## 目标

消除当前依赖审计中的高、中、低风险问题，加固认证、BFF、helper 和生产部署边界，同时保持现有页面的视觉风格、布局与内容不变。

## 范围

- 两阶段升级 Astro：已先升级到 6.4.8，再完成升级到 7.0.7。
- 为 Astro 7 显式保留 unified Markdown/MDX 渲染流程，避免文章、笔记和项目正文改变。
- 收紧共享密码认证、BFF 路由、helper CORS 和反向代理部署配置。
- 增加 GitHub Actions / Dependabot 配置与长期安全维护规则。
- 采集和比较桌面、移动端、深色模式页面截图。

## 不做的事

- 不重构 Astro/React/Node 分层，不替换设计系统，不重写现有页面。
- 不启用公网服务，不连接 ECS，不写入任何真实凭据。
- 不安装 Strix，不引入 CMS、数据库或重型安全依赖。

## 验收结果

- `pnpm audit --prod`：通过，无已知漏洞。
- `pnpm build`：通过；锁文件可由 `pnpm install --frozen-lockfile` 复现。
- 未登录、错误来源和未知路径请求：通过 API 冒烟检查。
- helper 代码默认仅限回环访问；生产 Nginx 和 ECS 端口仍需上线后验证。
- Astro 7 回归截图：桌面/移动端页面已生成，首页、笔记页、关于页已人工检查，未发现视觉回归。
